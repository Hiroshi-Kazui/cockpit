# ADR-0010: 目的完了時の評価パイプライン

- 日付: 2026-07-23
- 状態: proposed（`/cockpit-build M9` 起動＝プラン承認で accepted）
- 関連: spec §2 / §4.2 / §4.6 / §5、milestones/M9-purpose-evaluation/（起案元プラン）
- 先行: ADR-0008（出力先設定のプローブ検証・silent failure 禁止の型）、TD-5（Windows spawn とインジェクション不変条件）

## 文脈

ユーザー要望「目的達成ごとの評価（順調度・ストレス度・コミュニケーションコストの3軸）、
カテゴリ付き改善案、週次/月次遷移と総合評価、評価の出力先選択」を受け、
アーカイブ済みセッションの事後分析機能を追加する。spec §2 の「判断はアーカイブされた
ユーザ発言の内容から事後に行う」を実装する位置づけ。制約: 会話への介入・加工はしない、
append-only・元 JSONL 非改変、ペイン操作を評価がブロックしない、silent failure 禁止。

## 決定

- **D-1（トリガ）**: 評価は目的の「完了」操作成功後に fire-and-forget で起動する。
  入力はアプリ管理のアーカイブ（スプール）と SQLite のみから読む。pty・claude 管理下の
  元 JSONL には触れない。完了操作の応答は評価を待たない。
- **D-2（エンジン）**: titleGenerator と同型のヘッドレス `claude -p --model <model>` 1ショット。
  プロンプトは stdin 渡しのみ（argv に動的文字列を載せない — TD-5 のインジェクション不変条件）。
  応答は JSON を要求し、寛容パーサで検証する（未知フィールド無視・欠落許容・0–100 クランプ）。
  既定モデルは haiku（タイトル生成と同じ経済性優先）。`app_settings.evaluation_model` で変更可、
  `evaluation_enabled` で機能ごと無効化可。会話用 pty とは完全に別プロセスで、対話 UX に干渉しない。
- **D-3（極性）**: DB は要件の語彙どおりの生値を保存する（ストレス度・コミュニケーションコストは
  高いほど悪い）。レーダー表示のみ「面積が大きい＝良い」へ正規化（100−x）して描画し、
  生値は詳細に併記する。保存と表示の責務を分け、後日の集計・突合が要件の語彙で行えるようにする。
- **D-4（保存）**: 新テーブル `evaluations`。append-only であり、再実行は新規行の INSERT、
  行の削除・スコア書換の経路を設けない（UPDATE は status/report_state の遷移のみ）。
  目的ごとの現行評価は最新行。評価履歴自体もハーネス素材として残す。
- **D-5（出力先）**: `app_settings.evaluation_output_root`（任意・アーカイブミラーの root とは独立）。
  評価確定ごとに Markdown＋JSON レポートを temp+rename で書き出す write-through。
  レポートは SQLite から常時再生成可能な派生物のため、archive_mirror 型の resume/backfill 機構は
  設けない（小さな単発ファイルに差分同期は過剰）。設定時のプローブ検証・失敗の UI 可視化は
  ADR-0008 D-5 と同型。書き出し失敗は評価を失敗させず `report_state='error'` として可視化する。
- **D-6（集計）**: 週=ISO 週（月曜開始）・月=暦月、いずれもローカル時刻基準。バケット化・平均・
  総合は `shared/` の純関数（タイムゾーンオフセット引数で決定的にテスト可能）。ok 行のみ集計対象。
- **D-7（チャート）**: チャートライブラリの新規依存は追加しない。3軸レーダー・折れ線とも
  React＋SVG の手書き純コンポーネント。依存追加は ABI・供給網リスクに見合わない規模。
- **D-8（入力構築）**: transcript 全文を LLM に送らない。純関数がユーザ発言全量優先・
  アシスタント発言は先頭/末尾抜粋・総量上限で決定的に構築し、切り捨ての事実は
  `input_stats_json` に記録する。実質空の入力は LLM を呼ばず `skipped` で確定する。

## 帰結

- アプリの責務は「起動・表示・記録」＋「記録の事後分析」へ拡張される（出荷時に spec §1/§2 を更新）。
  会話への不介入・記録の不変性は従来のまま
- 評価は LLM の主観判定であり単発値の精度は保証されない。価値の重心は週次/月次の遷移に置く
- 完了ごとに haiku 1ショット分のトークン消費が増える（無効化トグルで制御可能）
- 将来、評価軸の追加はレーダー描画・集計が N 軸対応の純関数である範囲で拡張できる。
  過去目的の一括バックフィル評価は本 ADR の範囲外（別途明示操作として検討）
